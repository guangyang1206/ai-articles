PART 1

🚨 到底发生了什么？

先说结论，不绕弯子：

OpenAI 已经在 ChatGPT 中上线了广告系统。
当你在 ChatGPT 聊天时，后台会根据你的对话内容，在模型回复的数据流中直接插入广告卡片。

这意味着几件事：

① 你和 ChatGPT 的对话内容会被用来做广告定向

② 广告不是弹窗，而是像模型回复一样自然地出现在对话流中

③ 点击后会被加密 token 追踪，cookie 有效期 30 天

④ 已经有真实广告主入驻：Grubhub、GetYourGuide、Canva、Aritzia 等

PART 2

👀 广告长什么样？

广告以 carousel_cards（走马灯卡片）的形式出现在对话中。每张卡片包含：

🏷️

品牌名 + Logo

advertiser_brand

📝

标题 + 描述

title + body

🖼️

广告配图

image_url

🔗

跳转链接

target.value

💡 真实案例

一位用户聊"北京旅游攻略（长城、故宫）"时，ChatGPT 回复中出现了 Grubhub 的广告卡片：
标题："Get Chinese Food Delivered"
描述："Satisfy Your Cravings with Grubhub Delivery."

没错——你聊北京旅行，它给你推中餐外卖。这不是巧合，是精准投放。

值得注意的是，广告配图和 Logo 都托管在 OpenAI 自己的 CDN（bzrcdn.openai.com），不是从广告主服务器加载。这意味着 OpenAI 完全控制了广告素材的分发和加载体验。

PART 3

🎯 怎么做到精准投放的？

研究人员用同一个账号，在 6 个不同话题的对话中，收到了 6 个不同品牌的广告：

对话话题 投放品牌
北京旅行（长城、故宫） Grubhub — 中餐外卖
北京旅游预订 GetYourGuide — 长城一日游
北京机票 Axel — 航班预订
NBA 季后赛 Gametime — 赛事门票
春季时尚潮流 Aritzia — 时尚服饰
效率工具 / 演示文稿 Canva — 设计工具

🤔 投放逻辑是什么？

同一个账号，不同话题 → 不同广告。
这说明广告投放至少是上下文定向（contextual targeting）——读取你当前的对话内容来匹配广告主。

目前没有证据表明系统会利用你的历史对话记录做投放（但也不能排除）。如果未来引入 Memory 跨会话数据做定向……那就是另一个级别的精准了。

PART 4

🔐 四层加密追踪链路

这是技术人最应该关注的部分。ChatGPT 的广告归因系统使用了 4 个独立的 Fernet 加密 token，形成完整的追踪闭环：

（Fernet = AES-128-CBC 加密 + HMAC-SHA256 完整性校验，每个 token 前 9 字节含时间戳）

Token ① ads_spam_integrity_payload

在 SSE 数据内传输，不出现在点击 URL 中。用于服务端校验广告点击是否被伪造。

Token ② oppref

出现在点击 URL 上，被 OAIQ SDK 写入 cookie __oppref（有效期 720 小时 / 30 天）。这是核心归因 token——它跟着你在商户网站的每一次后续操作。

Token ③ olref

与 oppref 一起出现在点击 URL，但 SDK 不存储它。推测用于 OpenAI 服务端的出站链接日志记录。

Token ④ ad_data_token

Base64 包装的 JSON 内含另一个 Fernet token。在 SSE 载荷中传输，推测用于服务端点击时刻的数据核对。

🔄 完整归因闭环（一张图看懂）

1. 你发消息 → ChatGPT 后端打开 SSE 流
2. SSE 流中夹带 single_advertiser_ad_unit 广告对象
3. 前端渲染为卡片，你点击 → 跳转到商户页面
4. URL 携带 oppref + olref 加密参数
5. 商户页面加载 oaiq.min.js（OAIQ SDK v0.1.3）
6. SDK 读取 URL 中的 oppref → 写入 cookie（30天）
7. 你在商户网站的后续浏览/购买 → SDK 上报事件到 bzr.openai.com
8. OpenAI 完成归因：知道哪个广告带来了哪个转化 ✓

🧪 一个有趣的细节：研究人员抓到一个 Home Depot 的广告点击，token 铸造时间是 11:30:08 UTC，浏览器实际打开商户页面是 11:31:43 UTC。点击延迟：95 秒。说明用户看到广告后，犹豫了一分半才点——非常真实的人类行为。

PART 5

🛡️ 普通人怎么屏蔽？

好消息是，这套系统的关键节点是已知的。你可以：

方法一：屏蔽广告域名

在你的广告拦截器（uBlock Origin 等）或系统 hosts 文件中添加：

bzrcdn.openai.com
bzr.openai.com

⚠️ 注意：bzrcdn 负责加载广告素材，bzr 负责上报追踪事件。屏蔽后广告卡片可能无法渲染。

方法二：定期清除追踪 Cookie

如果你已经点过广告链接，检查并删除这两个 cookie：

__oppref（归因追踪，30天有效）
__oaiq_domain_probe（域名探测）

方法三：使用 API 而非网页版

目前广告系统只在 chatgpt.com 网页/App 端观察到。通过 API 调用模型（GPT-4o 等）暂未发现广告注入。如果你是开发者或重度用户，用 API 是当前最干净的方式。

方法四：关注 open_externally 字段

广告数据中有个字段 open_externally: false，意味着点击广告默认在 ChatGPT 内置浏览器中打开（而非跳转外部浏览器）。这让 OpenAI 能进一步观察你点击后的导航行为。如果必须点，尽量复制链接到独立浏览器中打开。

PART 6

💭 怎么看这件事？

说完技术，聊聊观点。

📊 OpenAI 的商业压力

OpenAI 目前估值 3000 亿美元，但烧钱速度极快。训练和推理成本是天文数字。广告是除了订阅和 API 之外的第三条收入线。对于 7 亿月活用户中的免费用户群体，广告模式几乎是必然选择。

Google 走过的路，OpenAI 也在走。只不过，搜索结果里夹广告大家已经习惯了，AI 对话里夹广告……你习惯吗？

🧠 信任的代价

AI 对话和搜索有一个本质区别：人们对 AI 助手的信任度远高于搜索引擎。

你 Google 搜索时知道前几条是广告。但当你的 AI "助手" 在帮你规划北京旅行时突然推荐一个外卖 app——你能分清这是它的 "建议" 还是广告吗？

这就是问题所在：当广告和回答混在一起，信任会被稀释。

⚖️ 客观说一句

从技术实现上看，OpenAI 做得是比较克制的：
• 广告是独立数据类型，不是模型输出
• 有明确的品牌标识（不是伪装成回答）
• 使用加密 token 而非明文追踪

但"克制"和"可接受"是两回事。尤其是考虑到 ChatGPT Plus 用户每月 20 美金——如果付费用户也看到广告，那就完全是另一个性质了。

📌 总结

今天你需要记住的 5 件事

1. ChatGPT 已经在对话流中投放广告了（真的，不是玩笑）
2. 投放基于你当前对话内容的上下文定向
3. 点击后通过 4 层加密 token + 30 天 cookie 做转化追踪
4. 屏蔽 bzrcdn.openai.com 和 bzr.openai.com 可阻断广告
5. API 调用暂时是干净的，网页/App 端有风险
